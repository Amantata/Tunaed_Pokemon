# CLAUDE.md — 프로젝트 가이드 (에이전트용)

> 이 문서는 AI 에이전트가 코드를 작성할 때 참고해야 하는 프로젝트 요구사항과 규칙을 정리한 것입니다.
> 클라이언트의 요구사항은 개발 진행에 따라 추가될 수 있습니다.

---

## 1. 프로젝트 개요

- **프로젝트명**: Tunaed_Pokemon (어장식 포켓몬)
- **목적**: 포켓몬 배틀 시스템을 기반으로 한 확장 배틀 시뮬레이터 구현
- **기반 시스템**: "오레마스" 시스템 (어장식 포켓몬 TRPG 룰셋)

## 3. 기술 제약 및 고려사항

### 3.1 배포 방식
- **필수**: 단일 실행파일로 배포 (설치 과정 불필요)
- 크로스플랫폼 지원을 고려할 것
- 온라인 배틀 시 서버/브라우저 방식의 장단점 비교가 필요

### 3.2 기술 스택 선정 시 고려사항
- GUI 프레임워크: 실행파일 빌드가 용이한 프레임워크 선택 필요
- 온라인 기능: 실시간 통신 (WebSocket 등) 고려
- 관전 모드: 실시간 상태 동기화 필요
- 배틀 상태 직렬화: 저장/불러오기, Undo/Redo를 위한 상태 관리

### 3.3 이미지 시스템 고려사항
- 트레이너 및 포켓몬에 이미지 파일(PNG, JPG 등) 연결 및 저장
- 이미지 표시 컨텍스트: 도감(데이터 열람), 배틀 화면, 파티 편집 화면
- 이미지 파일 경로를 데이터에 포함시켜 직렬화/역직렬화 지원
- 이미지 미설정 시 기본 플레이스홀더 이미지 표시

## 4. 게임 시스템 요약 (docs 기반)

> 상세 규칙은 `docs/` 디렉토리의 각 문서를 참조하세요.
> 아래는 구현 시 핵심적으로 참고해야 할 시스템 개요입니다.

### 4.1 데이터 모델

#### 트레이너 (Trainer)
- **기본 정보**: 이름, 별칭, 출신, 경력
- **이미지**: 트레이너 이미지 파일 경로 (도감/배틀 화면 표시용)
- **자질 (4가지)**: 지시, 통솔, 육성, 능력
  - 랭크: E / D / C / B / A / AA / AAA / S
  - 제작 시 최대 34점(또는 26점)을 4자질에 분배
  - 랭크별 필요 점수: E=0, D=2, C=4, B=6, A=10, AA=14, AAA=20, S=26
- **지령 포텐셜**: 기본 4식 + 숙달/전개 지령 (자질 랭크에 따라 사용 범위 변동)
- **고유 포텐셜**: 트레이너 고유 능력
- 상세: `docs/trainer_data_template.md`, `docs/trainer_qualities.md`

#### 포켓몬 (Pokemon)
- **기본 정보**: 이름, 성별, 아인종 여부, 타입(1~2개), 특성, 레벨
- **이미지**: 포켓몬 이미지 파일 경로 (도감/배틀 화면 표시용)
- **능력치 6종**: 체력, 공격, 방어, 특공, 특방, 속도
- **종족치 랭크**: E(1~35) ~ S(200+)
- **기술**: 종족치 총합에 따라 4~8개
- **포텐셜**: 역할, 분류, 주인, 이명, 계제, 속별, 유대, 선제, 회피, 내성, 격, 범용, 부수, 특권, PT, 전용
- 상세: `docs/pokemon_data_template.md`

### 4.2 핵심 시스템 규칙

#### 자질별 효과 요약
| 자질 | 역할 | 핵심 효과 |
|------|------|-----------|
| 지시 | 전투 중 지시력 | 랭크 차이당 C+1, 지령의 종류/수 결정, 타임(묘수) 횟수 |
| 통솔 | 파티 관리 | 역할 임명, 유대 수, 종족치 총합 제한, PT포텐셜 부여 |
| 육성 | 포켓몬 성장 | 레벨 보정, 계제 한계, 극화 가능 여부, 기술머신 |
| 능력 | 고유/전용 포텐셜 | 고유/전용 포텐셜 개수 및 강도 |

#### 배틀 흐름 (턴 순서)
1. 턴 개시 시 처리
2. 지시 입력 (트레이너 포텐셜 선행 입력 포함)
3. 행동 순서 결정: 가장 먼저 > 상대보다 먼저 > 우선도+N > 속도 비교
4. 기술/교대 실행
5. 턴의 최후 처리
6. 턴 종료 시 처리: 데미지(상태이상/날씨 등) → 드레인 → 회복 → 교대

#### 데미지 계산 핵심
- 급소: 5세대 기준, C+N에 따른 확률, 크리티컬 2배(일부 3배)
- 타입 일치 보너스(STAB) 적용
- 상성: 18타입 상성표 기반
- 복합 타입 기술: 유리한 상성으로 데미지, 양쪽 타입 효과 모두 적용

#### 교대 시스템
- 임의 교대: 자발적 교대 전반
- 통상 교대: 기술 없이 교대 (임의 교대의 하위 분류)
- 강제 교대: 상대/필드에 의한 랜덤 교대
- 추격/유턴: 특수 교대 처리 규칙 있음

### 4.4 상태 이상 / 상태 변화

#### 상태 이상 (6종 + 혼란)
화상, 동상, 독, 맹독, 잠듦(최대 2체 제한), 마비, 얼음, 혼란(상태변화이면서 상태이상 취급)

#### 상태 변화 (다수)
가열, 충전, 냉각, 침수, 송전, 악인, 하품, 동면, 요격, 분노, 혼란, 침묵, 망각, 헤롱헤롱, 사슬묶기, 앵콜, 봉인, 도발, 트집, 분진, 죽음의 카운트, 길동무, 아쿠아링, 뿌리박기, 저주, 씨뿌리기, 불꽃속/물속/땅속/어둠속, 텔레키네시스, 꿰뚫어보기, 미라클아이, 폭탄, 마음의눈, 회복봉인, 조이기, 검은눈빛, 대타출동, 난동부리기, 변신, 미래예지

- 상세: `docs/기본 규칙.md`, `docs/기타 시스템 처리규칙.md`

### 4.5 소지품 시스템
- 파티 내 중복 불가 (쥬얼/플레이트는 다른 타입이면 허용)
- 금지 소지품: 진화의휘석, 약점보험, 돌격조끼, 파괴의유전자 등
- 오리지널 소지품 존재 (보코열매, 마쥬열매, 옐로카드 등)
- 상세: `docs/소지품목록.md`, `docs/기본 규칙.md`

### 4.6 포텐셜 텍스트 규격
- 조건 트리거, 효과 트리거의 표준 텍스트 형식이 정의됨
- 동등한 텍스트 처리 규칙 (예: "무시한다" = "무효화한다" = "관통한다")
- 확률 기준: 드물게(5~15%), 저확률(20~30%), 중확률(40~60%), 고확률(60%+)
- 상세: `docs/potential_text.md`

## 6. 구현 우선순위 제안

1. **데이터 모델**: 트레이너, 포켓몬, 포텐셜, 소지품의 데이터 구조 정의
2. **배틀 엔진 코어**: 턴 진행, 데미지 계산, 타입 상성, 행동 순서
3. **상태 관리**: 상태이상/상태변화 시스템, 배틀 상태 직렬화(저장/로드/Undo/Redo)
4. **포텐셜 시스템**: 조건 판정 + 자동 발동 + 스크립트 작성 기능
5. **GUI**: 배틀 화면, 파티 편집, 데이터 관리
6. **온라인 기능**: 서버 구조, 실시간 통신, 관전 모드
