# 개발 필요요소 정리 (재작성)

> 본 문서는 **분할본의 원문 2종만**을 근거로 재정리했습니다.
>
> - `docs/분할본/새로운어장의첫걸음_저장/part-*.txt`
> - `docs/분할본/어장식_포켓몬_추가요소_설명/part-*.txt`
>
> 아래 체크리스트는 구현 필요요소를 기능 단위로 묶은 개발 기준입니다.

> 용어 안내: 본 문서에서 말하는 **추가사항**은
> `docs/분할본/어장식_포켓몬_추가요소_설명/part-*.txt`에 명시된 `추가사항 => ...` 규칙을 의미합니다.
>
---

## 1) 데이터 모델

### 1.1 개체/배틀 기본 데이터
- [ ] 포켓몬 기본 정보(이름, 타입 1~2, 특성, 기술 목록, 소지품)
- [ ] 기술 데이터(분류/타입/위력/명중/대상/우선도/접촉 여부)
- [ ] 기술 대상 타입 구분(단일/전체/자신/아군 필드/상대 필드/전체 필드)
- [ ] 트레이너 데이터(자질: 지시/통솔/육성/능력)
- [ ] 파티 데이터(트레이너 + 엔트리 + 역할/분류/이명/유대 등)

### 1.2 상태 데이터
- [ ] 상태이상(화상/얼음/독/맹독/잠듦/마비) 저장
- [ ] 상태변화(대상 적용/자신 적용) 저장
- [ ] 능력변화(랭크) 저장
- [ ] 교대 시 초기화/유지 규칙을 상태별로 저장

### 1.3 포텐셜 데이터
- [ ] 포켓몬 포텐셜 카테고리(역할/분류/주인/이명/계제/속별/유대/선제/회피/내성/격/범용/특권/PT)
- [ ] 트레이너 포텐셜 카테고리(지령/고유/전용)
- [ ] **고유포텐셜(트레이너)과 전용포텐셜(포켓몬) 분리 저장**
- [ ] 전용포텐셜 최대 2개 저장 지원(추가사항 반영)

---

## 2) 배틀 엔진

### 2.1 턴 파이프라인
- [ ] 턴 시작 처리
- [ ] 지시/기술 선택
- [ ] 우선도 + 속도 기준 행동 순서 결정
- [ ] 기술 실행/교대 실행
- [ ] 턴 종료 처리

### 2.2 교대 규칙
- [ ] 임의 교대/강제 교대 처리
- [ ] 교대 시 상태변화/능력변화 초기화 규칙 적용
- [ ] 교대 시 유지되는 예외 상태(문서 명시 항목) 별도 처리
- [ ] 죽어내밀기/턴 종료 교대 등 진입 경로별 처리

### 2.3 명중/급소/데미지
- [ ] 명중 계산(명중/회피 보정 포함)
- [ ] 급소 확률(C 보정) 계산
- [ ] 타입 상성 및 배율 적용
- [ ] 강화(배수)와 상승(랭크) 분리 계산
- [ ] 방어계열 필드/포텐셜의 데미지 완화 적용

---

## 3) 상태이상/상태변화 시스템

### 3.1 상태이상
- [ ] 상태이상 부여/해제/중첩 불가 규칙
- [ ] 교대로 상태이상은 유지되는 규칙
- [ ] 타입/효과 기반 상태이상 면역 처리

### 3.2 상태변화
- [ ] 대상 적용 상태변화 처리(혼란, 씨뿌리기, 앵콜, 도발 등)
- [ ] 자신 적용 상태변화 처리(대타출동, 방어, 난동부리기 등)
- [ ] 대부분 교대 해제 규칙 + 예외 유지 규칙 처리
- [ ] 멘탈효과 하위분류 및 전용 해제수단(예: 멘탈허브) 처리

---

## 4) 필드/환경 시스템

### 4.1 전장 상태
- [ ] 자진 필드 효과(리플렉터/빛의장막/신비의부적 등)
- [ ] 설치기(압정뿌리기/독압정/끈적끈적네트/스텔스록 등)
- [ ] 날씨 상태(쾌청/비/싸라기눈/모래바람 등)
- [ ] 룸 상태(트릭룸/매직룸/원더룸)
- [ ] Terrain 계열(그래스/미스트/일렉트릭/사이코)

### 4.2 《필드효과》 분리
- [ ] 일반 「필드」와 《필드효과》를 별개 시스템으로 구현
- [ ] 이계/던전/스타디움/짐 계열 효과 구현
- [ ] 동시 1개 유지, 후발 효과 우선(덮어쓰기) 규칙 반영
- [ ] 통솔형/능력형 필드형성 충돌 우선순위 설정값 반영(추가사항)

---

## 5) 포텐셜 엔진

### 5.1 발동/해석
- [ ] 조건 기반 자동 발동
- [ ] 확률 표현(드물게/저확률/중확률/고확률) 공통 처리
- [ ] 발동 시점(T개시/행동 전/피격 시/턴 종료 시) 처리

### 5.2 특성/포텐셜 분리
- [ ] 특성 처리 스택과 포텐셜 처리 스택 분리
- [ ] 특성 변경 효과가 포텐셜에 직접 간섭하지 않도록 처리

### 5.3 기능확장
- [ ] 기능확장 기술 사용 경로 지원
- [ ] 기능확장 사용 시 구애 상호작용 규칙 반영
- [ ] 기술머신/포텐셜 출처 기반 기능확장 트래킹

---

## 6) 트레이너 자질/지령

### 6.1 지시
- [ ] 지령 계열(맞춰라/피해라/돌아와/버텨라 등) 처리
- [ ] 지시 랭크 기반 급소 보정 처리
- [ ] 급소 보정 방식 2종 선택 지원(랭크차 기반 vs 절대값 기반, 추가사항)

### 6.2 통솔/육성/능력
- [ ] 통솔 기반 역할/PT/유대 처리
- [ ] 육성 기반 계제/기능확장/단련 보정 처리
- [ ] 능력 기반 고유/전용포텐셜 운용 처리

---

## 7) 배틀 설정(추가사항 직접 반영)

- [ ] 방호포텐셜 적용 종류 선택
- [ ] 급소 관통 포텐셜 종류 선택
- [ ] 명중체감치 정상화 4단계 설정
  - [ ] 미적용
  - [ ] 완전히 같은 상황에서만
  - [ ] 같은 명중률 기술에만
  - [ ] 모든 기술에 연속 적용
- [ ] 고유포텐셜 충돌 우선순위 설정
- [ ] 체력(HP) 가변 적용 여부 설정
- [ ] `기술이 급소에 맞기 쉬워진다(C+1)`를 랭크상승이 아닌 지속 C+1 효과로 처리

---

## 8) 부가 구현 요소

### 8.1 데이터 관리
- [ ] 배틀 중 상태를 구조적으로 저장/복원 가능하게 설계
- [ ] 규칙 예외를 데이터 주도(테이블/플래그)로 관리

### 8.2 UI/운영 지원
- [ ] 복잡 규칙 확인을 위한 상세 로그 출력
- [ ] 턴/행동/발동 원인 추적 가능한 이벤트 로그

---

## 9) 우선순위 제안

1. 데이터 모델(상태/포텐셜/필드 분리)
2. 턴 파이프라인 + 교대 + 명중/데미지
3. 상태이상/상태변화 + 필드/날씨
4. 포텐셜 엔진 + 기능확장 + 구애 상호작용
5. 자질/지령 + 배틀 설정(추가사항 항목)
6. 로그/복원/검증 도구
