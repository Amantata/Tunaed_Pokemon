# C안 구현작업 계획 체크리스트

> 기준 결정: C안(선택형 폼 + 고급 코드 직접수정 병행)
>
> 범위 정책: 더블배틀(F-01의 더블)과 온라인(F-02/F-04/F-05)은 후속 단계로 이관하고, 단기 구현은 싱글 배틀 기반 핵심 기능 완성에 집중한다.

---

## 1) 결정사항

- [x] C안 채택 확정
- [x] 초기 구현 범위를 싱글 배틀 중심으로 제한
- [x] 더블배틀/온라인 기능은 후속 마일스톤으로 분리

---

## 2) 1차 구현(핵심 동작 완성)

### 2.1 배틀 코어
- [ ] 턴 파이프라인(시작/지시/행동순서/실행/종료) 안정화
- [ ] 명중/급소/데미지 계산 규칙 보강
- [ ] 교대(임의/강제/추격/유턴) 처리 정합성 점검

### 2.2 상태/필드
- [ ] 상태이상/상태변화 지속·해제 규칙 정리
- [ ] 날씨/지형/룸/장벽/설치기 상태 관리 통합
- [ ] 「필드」(지속 효과)와 《필드》(특수 경기장) 분리 규칙 검증

### 2.3 저장/이력
- [ ] 배틀 저장/불러오기(B-01) 기본 흐름 완성
- [ ] 턴 히스토리 기반 Undo/Redo(B-02) 완성
- [ ] 종료 상태(battle_over, winner_side) 일관성 검증

---

## 3) 2차 구현(C안 편의성 강화)

### 3.1 선택형 폼(기본 사용자 경로)
- [ ] 트레이너/파티/기술/특성 입력 폼 UX 정리
- [ ] 포텐셜 선택형 입력(목록 기반) 기본 경로 제공
- [ ] 잘못된 입력에 대한 검증/에러 안내 강화

### 3.2 고급 직접수정(숙련 사용자 경로)
- [ ] 고급 편집 진입점 분리(일반 폼과 충돌 방지)
- [ ] 스크립트/직접수정 시 검증 규칙 및 안전장치 적용
- [ ] 수정 이력과 되돌리기 연계
