# Tunaed_Pokemon
어장식 포켓몬 구현 실증


아래의 클라이언트 요구사항 및 설명이 있고, 추가될 수도 있음.
### 클라이언트 요구사항 목록

배틀 진행에 대해
1. 진행중인 배틀 상황을 저장하고 불러오는 기능
2. 이전 턴 진행상황을 저장하고 뒤로가기, 앞으로가기 하는 기능
3. 


최종구현목표에 대해
1. 포켓몬 배틀 시스템을 기반으로 한 확장 배틀 시뮬레이터.
2. 최종 목표는 싱글 및 더블배틀 6v6 구현, 온라인 배틀을 목표로 한다.
3. 혼자서 사용할 경우 별도의 설치 없이 실행파일만으로 실행가능해야한다.
4. 온라인배틀은 실행파일로 서버연결, 도메인 브라우저 사용 중 장단점을 비교하여 추천방안을 제시할 것.
5. 온라인 배틀시 실시간 관전자 모드가 가능하여야 한다.

파티에 대해
1. 파티는 트레이너와 파티원으로 구분된다.
2. 1파티는 일반적으로 6명의 파티원으로 구성된다. (경우에 따라 7, 8명으로 늘어날 수도, 5명 이하로 줄어들 수도 있다.)
3. 

포텐셜에 대해
1. 스크립트로 프로그램 내에서 새로운 포텐셜을 짤 수 있어야 한다.
2. 기존에 존재하는 포텐셜에서 선택할 수도 있어야 한다.
3. 조건 만족시 배틀 처리단계에서 자동으로 처리되어야 한다.
