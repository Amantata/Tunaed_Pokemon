# Tunaed_Pokemon
어장식 포켓몬 구현 실증

포켓몬 배틀 시스템을 기반으로 한 확장 배틀 시뮬레이터. "오레마스" 시스템(어장식 포켓몬 TRPG 룰셋)을 기반으로 합니다.

### 파티 시스템

| 번호 | 요구사항 | 상태 |
|------|----------|------|
| P-01 | 파티 = 트레이너 + 파티원으로 구성 | 미착수 |
| P-02 | 기본 트레이너 1명 + 6명 파티원, 경우에 따라 1~8명 가변 | 미착수 |

> **P-02 비고:**
> - 파티원이 몇 명이든 배틀은 기본적으로 6대6 풀배틀. 트레이너의 고유포텐셜에 따라 변경될 수 있음.
> - 파티원 = 포켓몬에서의 "포켓몬" 역할, 트레이너 = 포켓몬에서의 "트레이너" 역할.
> - 이 시뮬레이션은 포켓몬의 시스템을 따온 것으로, 포켓몬 배틀 시스템을 기반으로 한 것뿐이므로 인간 vs 인간 이능력 배틀과도 비슷함.

### 포텐셜 시스템

| 번호 | 요구사항 | 상태 |
|------|----------|------|
| PT-01 | **UI 폼(조건·효과 드롭다운) 중심**으로 새로운 포텐셜 작성 가능, 고급 효과는 스크립트로 보조 | 미착수 |
| PT-02 | 기존 포텐셜 목록에서 선택 기능 | 미착수 |
| PT-03 | 조건 만족 시 배틀 처리 단계에서 자동 발동 | 미착수 |
| PT-04 | 개체별 포텐셜(계제 1~4, 속별, PT1·2, 전용포텐셜, 고유포텐셜)은 **UI 폼(기본) + 스크립트(보조)** 방식으로 작성; 검증은 `docs/트레이너 예제.md` 데이터로 배틀 로그 재생 테스트 | 미착수 |
| PT-05 | 포텐셜 명칭 엄격 구분: **고유포텐셜** = 트레이너마다 다른 고유 능력 / **전용포텐셜** = 포켓몬의 특히 강한 포텐셜로 일반 도감으로는 알 수 없음 | 미착수 |
| PT-06 | 특성과 포텐셜은 독립적: 특성에 영향을 주는 효과는 포텐셜에 영향이 없고, 그 반대도 마찬가지 | 미착수 |
| AX-04 | 배틀 설정에서 **방호포텐셜 적용 종류**를 선택 가능해야 함 | 미착수 |
| AX-05 | 배틀 설정에서 **급소 관통 대상 포텐셜 종류**를 선택 가능해야 함(기본: 완화계) | 미착수 |
| AX-06 | 데이터 모델에서 포켓몬 1개체당 전용포텐셜을 최대 **2개**까지 지정 가능해야 함 | 미착수 |
| AX-08 | 고유포텐셜 충돌(예: 능력형 이계 vs 통솔형 필드형성) 시 배틀 설정에서 **우선순위**를 지정 가능해야 함 | 미착수 |

> **PT-05 명칭 규칙:**
> - **고유포텐셜**: 트레이너마다 다른 고유 능력. 모든 트레이너가 각자 다른 고유포텐셜을 가짐.
> - **전용포텐셜**: 포켓몬의 특히 강한 포텐셜로, 일반 도감으로는 알 수 없음.
>
> 이 두 항목은 데이터 모델에서 별도의 명확한 필드로 구현해야 함.
>
> **포켓몬(파티원) 포텐셜 명칭 전체 목록:**
> 【포텐셜】 『역할』, 『분류』, 『주인』, 『이명』, 『계제 ①』, 『계제 ②』, 『계제 ③』, 『계제 ④』, 『속별』, 『유대』, 『선제』, 『회피』, 『내성』, 『○격』, 『범용』, 『부수』, 『특권』, 『PT ①』, 『PT ②』, 전용포텐셜
>
> **포텐셜 보유 규칙:**
> - 모든 포켓몬이 항상 보유: 계제①~②, 속별, 선제, 회피, 내성, 격, 범용
> - 파티 배분 및 단련도에 따라 가변: 역할, 분류, 주인, 이명, 계제③~④, 유대, 부수, 특권, PT①~②

## Windows 무설치 실행 방법 (F-03, ADD-02)

### 사용자(설치 없이 실행)

1. GitHub **Releases**에서 `TunaedPokemon-portable-windows.zip` 다운로드
2. 압축 해제
3. `TunaedPokemon.exe` 실행

- Python/Qt 등 추가 프로그램 설치가 필요 없습니다.
- 사용자 데이터 저장 위치: `%APPDATA%\TunaedPokemon\`

### 개발자(로컬에서 직접 빌드)

**사전 준비**: Python 3.11+ 설치 (https://python.org)

```bat
build_windows.bat
```

결과물: `dist\TunaedPokemon.exe`

**수동 빌드**:
```bat
pip install -e ".[build]"
pyinstaller tunaed_pokemon.spec --noconfirm
```
